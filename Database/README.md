The original database was designed and implemented using Supabase (PostgreSQL + PostGIS) because it provides native support for spatial databases, making it the most suitable choice for performing geographic analysis and managing location-based data.

During development, all database tables, relationships, and spatial analysis were implemented on Supabase.

However, due to backend integration challenges and the limited project timeline, the running version of the application used a local MySQL database for demonstration purposes. The database schema was migrated to MySQL to allow the system to operate during the final presentation.

From a technical perspective, Supabase (PostgreSQL + PostGIS) remains the more appropriate database solution for this project because of its built-in spatial capabilities and support for advanced GIS operations.
